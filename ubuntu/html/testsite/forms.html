URLs with multiple = must result in a correct form (caused problems in previous versions):
data2html.asp?login/=user=Guest

A URL  query string with hex encoded binary data must be kept in the URL:
data2html.asp?login/=user=Gu%00est

A URL query string with invalid hex encoding must also be kept in the URL:
data2html.asp?hexdata=%65%66%67%0%
